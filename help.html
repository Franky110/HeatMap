Trip Manager - Help

Trip Manager imports your recorded trips (from GPX files, Komoot, Strava or Garmin Connect) and displays them as a combined heatmap, showing which routes you travel most often. Routes used many times appear brighter because multiple overlapping lines stack on top of each other. Click anywhere on the map to find all the trips that passed through that spot.

0. Installation

Trip Manager is a set of files you can copy anywhere on your computer (e.g. a folder on your Desktop) — there is no traditional installer.

  1. Double-click setup.bat. The first time it runs, it will:
    • check whether Python is installed, and if not, try to install it (or open the download page so you can install it manually — during install, make sure "Add python.exe to PATH" is checked);
    • create a private env folder with the packages Trip Manager needs, without affecting any other Python on your computer.
    If it tells you Python was just installed, close the window and run setup.bat again.
  2. Once setup finishes, double-click Trip Manager.bat to start the program. You can create a shortcut to this file on your Desktop for convenience.
Setup only needs to be run once (or again after updating Trip Manager to a new version). After that, just use Trip Manager.bat to launch it.

1. Importing data

Trip Manager works with GPX files. There are several ways to get them:

Import from Komoot / Strava / Garmin Connect

Use the import buttons in the top-left of the window. The first time you use a service you'll be asked for your login credentials. For Komoot these are stored encrypted on your computer and won't be asked again on the same PC.

Add files manually

ButtonWhat it does
Add files...Pick one or more .gpx files to add.
Add folder...Pick a folder; every .gpx file inside it is added.
Remove selectedRemoves the highlighted trip(s) and deletes the GPX file.
RefreshRe-scans your data folder and reloads the trip list.

2. Processing your trips

"Processing" reads each GPX file, decimates the GPS track to remove near-duplicate points, and writes the result to the output folder so the heatmap can display it. Processing is fast — a typical batch of trips takes a few seconds.

Set any processing options you want, then click Run processing. A progress bar and log show what's happening. When processing finishes, a dialog offers to open the visualization right away.

Processing writes raw_data.js and a trip_details/ folder into the Output folder. The heatmap page reads these files.

3. Processing options explained

OptionWhat it does
Limit (0 = all) Maximum number of trips to include, starting from the most recent. Set to 0 to include every trip. Useful for a quick test before processing your entire history.
Start date / End date Only include trips recorded between these dates (YYYY-MM-DD). Leave blank for no limit on that side.
Min distance (km) Skip trips shorter than this distance — handy for filtering out accidental short GPS traces.
Sport Only include trips of the selected sport (Bike, Run, Walking, etc.). Choose All to include every sport.
Only process new trips When checked, trips already present in the output are skipped, and new results are merged into the existing output. This is much faster than reprocessing everything when you just want to add a few new trips.
Output folder Where the processed data is written. Defaults to the processed folder inside your Trip Manager data folder.

4. Viewing the heatmap

Once you have processed at least one trip, click Open visualization. This starts a small local web server and opens the heatmap in your browser.

The map shows every trip as a solid line. Routes you have traveled many times appear brighter because multiple semi-transparent lines overlap — no preprocessing needed. The color of each line reflects its sport type.

5. Heatmap page options

ControlWhat it does
Select data folder... Loads data from a different folder — for example, one someone else shared with you. The label below shows which folder is currently loaded.
GPS tracks (checkbox + Width / Opacity sliders) Shows or hides all trip lines. Width controls line thickness; Opacity controls how transparent each individual line is. Lower opacity makes overlapping routes stand out more clearly.
Sport type filters One checkbox per sport found in your data. Untick a sport to hide those trips. Click the color swatch to change the color used on the map.
Click radius The search radius (in metres) used when you click the map. All trips that have a GPS point within this radius of your click are listed in the sidebar.

Clicking the map

Click anywhere on the map. A translucent circle shows the search area, and the sidebar lists every trip whose GPS track passes within that radius. The header shows the count: "X trips within 50 m". Click any trip in the list to open its dashboard.

6. The trip dashboard

Clicking a trip in the sidebar opens the dashboard at the bottom of the screen, showing elevation and speed profiles for that trip, and highlights the route faintly on the map. The speed axis is scaled to the sport type (e.g. tighter for Running than for Biking). GPS noise spikes are filtered automatically with a 5-point median smoother.

ControlWhat it does
X: Distance / Time Toggles the horizontal axis between distance (km) and elapsed time (min).
Compare Enters comparison mode — click the map or pick from the trip list to overlay a second trip on both charts.
Back to list Closes the dashboard and returns to the nearby-trip list in the sidebar.
Close Closes the dashboard, clears all highlights, and hides the sidebar.

Selecting part of a trip

There are two ways to select a segment of the trip:

  • On the chart: two vertical red lines mark the start and end of the selection, initially covering the whole trip. Drag either line (cursor changes to ↔ when you hover near it) to focus on a segment.
  • On the map: click and drag directly along the highlighted trip line to select a segment. The cursor changes to a crosshair during drag.

The portion of the chart outside the selection is dimmed, the selected segment is shown as a thick black line on the map, and all statistics update live.

StatisticMeaning

DurationTotal elapsed time of the selection.

Avg speedAverage speed over the whole selected duration, including stops.
Moving avgAverage speed counting only the time spent moving faster than 1 km/h.

DescentTotal elevation lost (metres) within the selection.

Avg power / Max powerAverage and peak power output in watts within the selection (only shown when power data is present — e.g. Garmin rides with a power meter).

Comparing two trips

Click Compare in the dashboard header to enter comparison mode. The sidebar prompts you to pick a second trip — click the map to find nearby trips, then click one in the list. The second trip's elevation and speed are overlaid on the charts in orange, with a legend, and a summary bar below the charts shows key stats for both trips side by side. Click Compare again (now labelled "Cancel compare" or the button toggles off) to clear the comparison.

7. Segments

Segments let you define a named stretch of road or trail and instantly see a leaderboard of every trip that has passed through it, ranked from fastest to slowest — similar to Strava segments, but entirely local.

Creating a segment

  1. Click ◆ Segments in the top-right of the heatmap page.
  2. Click + Create segment in the panel that opens.
  3. Click the start point on the map — a green marker appears.
  4. Click the end point on the map — a red marker appears.
  5. Type a name for the segment and click Save.
Segments are saved in your browser's local storage. They persist across page reloads but are browser-specific — they won't transfer to another device or browser automatically.

Selecting a segment

Click a segment card in the panel. The segment markers appear on the map and a leaderboard opens below the card, listing every trip that passes through the segment within an 80-metre match radius. Results are ranked by the time taken to travel from the start point to the end point. Click any entry in the leaderboard to open that trip's dashboard, with the chart selection automatically snapped to the segment portion.

Segment operations

ButtonWhat it does
RenameChange the segment's name.
ReverseSwap start and end points — useful for a segment you also travel in the opposite direction.
CopyDuplicate the segment (useful as a starting point for a slight variation).
DeletePermanently remove the segment and its leaderboard.

Comparing two efforts on a segment

With a segment selected and its leaderboard visible, click Compare in the dashboard header. The leaderboard switches to "pick an effort" mode — click any other entry to overlay that trip on the charts, with both efforts aligned to the segment portion only. The compare bar below the charts shows key stats for each effort side by side. Click Compare again to exit comparison mode.

8. Data Wizard

The Data Wizard is an advanced tool for inspecting and cleaning up your trip library. Open it from the Data Wizard… button in the main Trip Manager window.

The trip list

The wizard shows every trip it finds in your data folder as a table with the following columns:

ColumnMeaning
DateDate extracted from the filename (YYYY-MM-DD).
NameThe GPX filename.
SportDetected sport category (Bike, Run, Walking, …).
SourceImport source (garmin, komoot, strava, or blank for manually added files).
SensorsSensor data present in the raw GPX — any combination of HR (heart rate), Cad (cadence), Pwr (power), Tmp (temperature). Blank if no sensor data is recorded.
DistanceTotal GPS track distance in km (from the raw GPX).
StatusRaw only — GPX file present but not yet processed; Processed only — processed data exists but the raw GPX has been deleted; Both — both files present.

Filters

Use the filter bar above the list to narrow down what is shown: status, sport, date range, minimum distance, and a text search on the filename. The count line below the filter bar shows how many trips are displayed vs. total.

Find duplicates

Click Find duplicates to scan for trips that were imported more than once. Two trips are considered duplicates when they share the same date and their GPS distances are within 5% of each other. Duplicate pairs are highlighted with distinct background colors so each pair is immediately recognisable: pair 1 in yellow, pair 2 in green, pair 3 in blue, and so on. A summary dialog lists the pairs found.

Deleting files

ButtonWhat it does
Delete raw GPXPermanently deletes the raw .gpx file for selected trips that have one. Use this to free disk space once a trip is processed and you no longer need the original.
Delete processedRemoves the processed output for selected trips. The raw GPX is kept. Re-run processing to rebuild the output.
Deletions are permanent. The Data Wizard does not have an undo function. Use Find duplicates first to identify which copy to keep before deleting.

9. Where your data is stored

The first time you run Trip Manager, it asks you to choose or create a data folder. Everything personal — your GPX files, processed data, and service credentials — is stored there, separate from the program itself. This means you can share or update the program without sharing your trip data, and you can point different copies of the program at the same data folder.

Inside your data folder you'll find:

  • raw_gpx/ — your imported GPX files
  • processed/ — the output of processing, read by the heatmap page
  • komoot_credentials.json — your encrypted Komoot login (if used)
